Custom Mass Emailing Program

Situation: A Genesistems client wanted to complete their billing process electronically to save on postage. My supervisor was trying to implement a email notification system with the K-Print service (using KCML). The service did not print out the appropriate PDF template while using its email features.

Solution: After a few weeks I made a proposal to create an email module that could be completed relatively quickly using Visual Basic.NET. It could be run by a user or without a GUI as a service. Project Description: When set to ‘Automatic’ the program looks in a directory every set amount of seconds for a new XML document generated from the KCML system. It then commences to send out the emails using the default settings unless the XML sets special values (i.e. using a different email, subject line, etc.). If the email has an attachment, the XML contains its location and the email program attaches the file to the message. When utilizing the User Interface a user can screen the list of emails, and if necessary, stop the program from sending the selected email(s). The program also contains a test button to send a single email with the base settings specified. In addition I created a Visual Basic program for each individual user’s computer that allows them to read the archived XML lists on the server and view sent emails for verification. The program is currently being used in a veterinary clinic’s accounting department, and for a high school’s sport notification system.

Technology Used: Visual Studio 2010, Visual Basic.NET, XML, Simple Mail Transfer Protocol (SMTP)

Morrisville Final Exam scheduler

Situation: Currently the Registrar at Morrisville State College makes the final exam schedule by hand. This requires many hours of trial and error as regular class times have no method that can be followed without creating conflicts with other classes in student’s schedules. The capstone Application Software class (CITA 450) was tasked with creating an automated solution based on query results from the master course database, written in CSV (Comma-Separated Values) files.

Solution: The program imports a master list of classes, and assigns each to a course object. Then it imports the Conflict Matrix and assigns the conflicting courses to the associated course object, which has a parameter that is a list of courses it conflicts with. The Conflict Matrix is a list of courses that have students enrolled in both. For example ‘CITA197, BSAD123, 3′ indicates that three students are enrolled in both CITA197 and BSAD123. It then loads all the courses into a list View. Clicking on a course shows a list of courses that don’t conflict, and a list of courses that conflict with the selected course.

Technology Used: Visual Studio 2010, Visual Basic.net, Comma-Separated Values
